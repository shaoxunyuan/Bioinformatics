一、在 NCBI 上运行 BLAST

以 demo_dna_1 为例演示 BLASTn；流程对 BLASTp 也完全类似，只是选择的程序与数据库不同。

步骤 A · 打开 NCBI BLAST 页面
  1. A1 · 访问网站 在浏览器打开：
    https://blast.ncbi.nlm.nih.gov/Blast.cgi
  2. A2 · 选择程序 对 DNA 序列：点 nucleotide BLAST（BLASTn）； 对蛋白序列：点 protein BLAST（BLASTp）。
  3. A3 · 粘贴查询序列 在 “Enter Query Sequence” 文本框中粘贴上方的 demo_dna_1 FASTA。
步骤 B · 选择数据库与参数
  1. B1 · Database 教学演示可选择：
    • nt：所有核酸序列（较慢，命中多）；
    • 或一个较小的 genome / RefSeq 库，速度更快。
  2. B2 · Program Selection 保持默认（megablast）即可；如果希望找到更远的同源，可以选择 discontiguous megablast。
  3. B3 · 其他设置 初学者可以先保持默认，只确认： Expect threshold：10 或 0.05 Max target sequences：50–100
  4. B4 · 提交任务 点击页面底部的 BLAST 按钮，等待队列运行完成。

NCBI 结果页面怎么看？

NCBI BLAST 的结果可分为三块：概要（Graphic summary）、hits 表格（Descriptions）、以及具体比对详情（Alignments）。

C · Hits 表（Descriptions 区）

这个表格是 “命中列表”，每一行代表一条满足条件的 subject 序列。

Score
Score
比对得分，越高说明越匹配。
E-value
E
期望值，越小越显著，10⁻³ 以下通常认为有意义。
Query cover
%
命中覆盖了 query 的多少百分比。
Ident
%
相同碱基/氨基酸的百分比（percent identity）。

选择一条 高 coverage + 高 identity + 低 E-value 的记录，点击 accession 或 “Description” 进入比对详情。

D · Alignments 区（具体比对）

在 Alignments 部分，你会看到类似下面的三行结构：

Query 1 ATGGTGCTGCTGCTGCTGC---TGCTGCTGCTGCTGCTGCTG 45 |||||||||||||||||| ||||||||||||||||| Sbjct 10 ATGGTGCTGCTGCTGCTGCTGCTGCTGCTGCTGCTGCTG 54

阅读要点：

  • Query / Sbjct：分别是查询序列与数据库中的序列。
  • 中间行符号：| 表示完全相同，空格/其他符号表示 mismatch。
  • 缺口（-）：代表插入 / 缺失（indel）。

二、在 CNGB 上运行 BLAST

CNGB（国家基因库）提供针对本地数据资源的 BLAST 服务，界面与 NCBI 类似，但数据库多为国人自己构建的数据集。

步骤 E · 进入 BLAST 工具
  1. E1 · 登录 CNGB 在浏览器访问 CNGB 主站，登录个人账号（部分数据或工具需要登录后使用）。
  2. E2 · 打开 BLAST 页面 通常在 “Tools / 工具” 或 “Sequence Analysis / 序列分析” 模块中找到 BLAST 入口。
  3. E3 · 选择程序 & 数据库 根据你的需求选择：
    • BLASTn / BLASTp 等程序类型；
    • 相应的 本地数据库（如某个物种基因组、转录组、宏基因组项目等）。
步骤 F · 粘贴序列并运行
  1. F1 · 输入查询序列 将上文的 demo_dna_1 或 demo_protein_1 粘贴到 query 文本框，保持 FASTA 格式。
  2. F2 · 设置阈值 类似 NCBI，可设置 E-value 阈值、最大返回条数。教学示例可以使用默认值。
  3. F3 · 运行与查看 点击 “Run / Submit” 按钮。任务完成后，CNGB 一般也会给出：
    • 命中列表（含 E-value、identity、coverage）；
    • 每条 hit 的比对详情（alignment，可下载或导出）。
    阅读方法与 NCBI BLAST 基本相同。
何时优先选择 CNGB BLAST？
如果你的研究数据来自某个特定 CNGB 项目（比如大规模队列、特定物种资源库）， 或者你想优先匹配国内已经整理好的数据库，那么在 CNGB 上跑 BLAST 会比 NCBI 更有针对性。

三、如何系统解读 BLAST 结果

不论是在 NCBI 还是 CNGB，BLAST 的核心解读逻辑完全一致：看的是 “相似程度” 与 “统计显著性”。

关键指标速记
Query cover
覆盖度
命中序列覆盖 query 的比例。越高越好；低于 50% 时通常需要谨慎解读。
Percent identity
一致性
碱基/氨基酸逐位相同的百分比。> 90% 多为高度同源，70–90% 可能为同源/近源。
E-value
显著性
越小越好，10⁻³ 以下常被认为是可靠 hits；功能注释时会更严格。
Gaps / mismatches
变异位置
可以帮助你定位突变、保守区和潜在的结构/功能域差异。

对于教学示例（短序列），你可以尝试：

  • 先用 demo_dna_1 比对，记录 top hit 的 identity / coverage / E-value；
  • 再把 demo_dna_2 作为 query，比对结果与上一次对比，体会少量碱基变化对指标的影响。

四、常见问题（FAQ）

初学者在用 BLAST 时最常遇到的几个问题，可以在课堂或自学时直接对照排查。

Q1 · 结果一片空白，没有任何 hits？
  • 确认是否选择了适合的数据库（不要选错类型，如用蛋白去查核酸库）。
  • 检查 query 是否为有效序列：是否包含非法字符、过于短（< 15 bp）或全是 N。
  • 尝试放宽 E-value 阈值或增加最大返回序列数。
Q2 · 命中很多，但描述五花八门，看不懂？
  • 优先关注：已注释的基因 / 蛋白（description 中含 gene/protein 名称的）。
  • 按 E-value 递增、identity 递减排序，选前几条看 alignments 是否合理。
  • 如果 hits 多为 “hypothetical protein”，可能说明该区域功能尚不明确。
Q3 · NCBI 与 CNGB 结果不一致？
  • 两者使用的数据库不同：一个是全球公开数据，一个是特定本地库。
  • 同一条序列在不同库中可能有不同的注释版本。
  • 比较时请记录：数据库名 + 版本 + 运行日期。
Q4 · 如何在报告/作业中描述 BLAST 结果？
  • 说明所用网站（NCBI / CNGB）、程序类型（BLASTn / BLASTp）与数据库。
  • 给出 top hit 的 accession、物种名、identity、coverage、E-value。
  • 可附上一段 alignment 截图或文本，标出关键突变位点。

1. 两两序列比对（全局 / 局部）

全局比对（Needleman–Wunsch）适合整体同源、长度相近序列； 局部比对（Smith–Waterman）适合在长序列中找最高相似片段（保守结构域/突变邻域）。 R 中可用 Biostrings::pairwiseAlignment() 直接完成，并支持替换矩阵与空位罚分。

1.1 DNA：全局 vs 局部

dna1 <- DNAString("ACGTTGACCTGAACTGAC")
dna2 <- DNAString("ACGTCGACCTGA---GAC") # 这里故意放入缺口仅用于说明（真实数据一般先不带'-'）

# 实际比对时输入应为不带 '-' 的原始序列，这里先清理缺口
dna2_raw <- DNAString(gsub("-", "", as.character(dna2)))

aln_global <- pairwiseAlignment(
  pattern = dna1,
  subject = dna2_raw,
  type = "global",
  substitutionMatrix = nucleotideSubstitutionMatrix(match = 2, mismatch = -1, baseOnly = TRUE),
  gapOpening = -5,
  gapExtension = -2
)

aln_local <- pairwiseAlignment(
  pattern = dna1,
  subject = dna2_raw,
  type = "local",
  substitutionMatrix = nucleotideSubstitutionMatrix(match = 2, mismatch = -1, baseOnly = TRUE),
  gapOpening = -5,
  gapExtension = -2
)

score(aln_global)
score(aln_local)
aln_global
aln_local

3. 一致性 / 相似性量化

常用指标包括：percent identity（PID，一致性）、以及基于替换矩阵的相似性（蛋白更常用）。 这里用 Biostrings::pid() 对 MSA 直接算两两 PID 矩阵。 

# 常用口径：忽略 gap 的 percent identity（PID, %）
pid_matrix_from_aligned <- function(aligned_set, ignore_gaps = TRUE) {
  aln_strings <- toupper(as.character(aligned_set))
  x <- do.call(rbind, strsplit(aln_strings, ""))
  n <- nrow(x)
  out <- matrix(NA_real_, nrow = n, ncol = n, dimnames = list(names(aln_strings), names(aln_strings)))

  for (i in seq_len(n)) {
    for (j in seq_len(n)) {
      a <- x[i, ]
      b <- x[j, ]
      keep <- if (ignore_gaps) (a != "-" & b != "-") else rep(TRUE, length(a))
      denom <- sum(keep)
      out[i, j] <- if (denom == 0) NA_real_ else 100 * sum(a[keep] == b[keep]) / denom
    }
  }
  out
}

aligned_dna_set <- msa_dna  # AlignSeqs 的返回本身就是 DNAStringSet（带 gap）
pid_dna <- pid_matrix_from_aligned(aligned_dna_set, ignore_gaps = TRUE)
round(pid_dna, 2)

4. 进化距离矩阵

距离矩阵是构树的数值基础。DNA 常见模型：Jukes-Cantor、Kimura 2-parameter（K80）。 这里用 phangorn 将对齐结果转换为 phyDat，再计算距离。 

# ape::dist.dna 支持 K80（Kimura 2-parameter）
aligned_dna_set <- msa_dna
aln_strings <- tolower(as.character(aligned_dna_set))
aln_strings <- gsub("-", "n", aln_strings)  # gap 视作未知 n，便于距离计算

m_char <- do.call(rbind, strsplit(aln_strings, ""))
rownames(m_char) <- names(aln_strings)
dnabin <- ape::as.DNAbin(m_char)

dist_k80 <- ape::dist.dna(dnabin, model = "K80", pairwise.deletion = TRUE)
as.matrix(dist_k80)

5. 变异位点（SNP / Indel）

下面给出一个“按列扫描对齐”的实用做法：找出有变异的列、区分是否包含缺口（Indel），并输出位点表（可用于下游筛选高变异区域）。 

variant_sites <- function(aligned_set) {
  aln_strings <- toupper(as.character(aligned_set))
  m <- do.call(rbind, strsplit(aln_strings, ""))  # rows = sequences, cols = alignment positions
  col_is_var <- apply(m, 2, function(x) length(unique(x)) > 1)
  col_has_gap <- apply(m, 2, function(x) any(x == "-"))
  idx <- which(col_is_var)

  data.frame(
    pos = idx,
    has_gap = col_has_gap[idx],
    pattern = apply(m[, idx, drop = FALSE], 2, function(x) paste(x, collapse = "")),
    stringsAsFactors = FALSE
  )
}

aligned_dna_set <- msa_dna
var_tbl <- variant_sites(aligned_dna_set)
head(var_tbl, 10)

7. Motif / 保守片段检索

Biostrings 支持在 DNA/蛋白序列上做快速模式匹配（精确/模糊）。 示例：在一组序列中查找 motif “ACCTG” 出现的位置。 

motif <- DNAString("ACCTG")
hits <- lapply(dna_set, function(x) {
  h <- matchPattern(motif, x)
  if (length(h) == 0) return(data.frame(start = integer(), width = integer()))
  data.frame(start = start(h), width = width(h))
})
hits

FAQ：常见报错与排查

  • Bioconductor 包装不上 / 下载慢：重试即可；必要时切换网络或设置代理。也可先跑脚本，它会自动补齐缺失包。
  • PowerShell 里运行 R --version 失败：这是因为 R 常被当作历史命令别名；用 Rscript --version 或 & "C:\\...\\R.exe" --version。
  • 对齐对象维度不一致：不同包的对齐对象类型不同（DNAStringSet/MultipleAlignment/Msa*），建议 class()、str() 先确认再转换。
